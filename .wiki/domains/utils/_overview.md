# Utils

## What this domain does
The `utils` domain is Django's foundational utility layer — a large collection of low-level, cross-cutting concerns that the rest of the framework depends on. It does not model any business concept; instead it provides the building blocks that every other domain consumes: safe string handling, lazy evaluation, translation dispatch, cryptographic primitives, CSP policy building, format localization, function introspection, and archive extraction. Nearly the entire codebase depends on it, making it the widest-reaching domain in the project. `[llm-analyzed]`

## Key behaviors
- **Translation backend is resolved lazily and cached per attribute**: The `Trans` class delays importing `trans_real` or `trans_null` until the first call to any translation function. Once a function is resolved, it is stored via `setattr`, bypassing `__getattr__` on all future calls for performance. A side-effect: changing `USE_I18N` after the first translation call has no effect unless all resolved attributes are manually deleted from `_trans.__dict__`. When `USE_I18N` is True, it also wires autoreload signals to watch translation files for changes. `[llm-analyzed]`
- **Plural-aware lazy strings defer number substitution until format time**: `ngettext_lazy` and `npgettext_lazy` use `lazy_number()` to create `NumberAwareString` instances. When the `number` argument is a key name (string) rather than an integer, the number is extracted from format arguments at render time via `__mod__` or `format()`. This means singular/plural selection happens at the point of string interpolation, not at definition time. The `KeyError` raised when the number key is missing has a specific error message explaining why it's required. `[llm-analyzed]`
- **CSP.NONCE is a sentinel placeholder, not a real nonce value**: The `CSP.NONCE` enum value (`"<CSP_NONCE_SENTINEL>"`) is explicitly documented as arbitrary — it should never appear in an actual HTTP header. The `build_policy()` function replaces it with `'nonce-{actual_nonce}'` if a nonce is provided, or silently removes it if not. This means a CSP config with `CSP.NONCE` but no nonce provided at build time will omit the nonce directive entirely without error. `[llm-analyzed]`
- **LazyNonce.__bool__ signals whether the nonce has been accessed, not whether it exists**: `LazyNonce.__bool__` returns `True` only if `_wrapped is not empty`, i.e., only after the nonce has been generated by being accessed. This is intentional: template code like `{% if csp_nonce %}` will be falsy until something else in the page has already triggered nonce generation. Third-party libraries that support optional CSP can use this to conditionally add nonce attributes without requiring CSP to be configured. `[llm-analyzed]`
- **salted_hmac always derives a key via hashing, even when unnecessary**: Even when `len(key_salt + secret)` is smaller than the hash block size (making the derivation redundant since HMAC would do the same internally), the key derivation step is always performed. The code comment explicitly acknowledges this redundancy and states it is kept for consistency — every call site must generate keys the same way regardless of input size. `[llm-analyzed]`

## Domain interactions
- **Conf (settings)**: Accessed lazily throughout utils — translation reads USE_I18N, crypto reads SECRET_KEY, formats reads FORMAT_MODULE_PATH and all FORMAT_SETTINGS. Most reads are deferred to avoid requiring settings to be configured at import time. `[llm-analyzed]`
- **Dispatch (signals)**: Translation wires autoreload_started and file_changed signals to watch for .po/.mo file changes when USE_I18N is True. This happens on first access to any translation function. `[llm-analyzed]`
- **Db / ORM**: make_hashable (hashable.py) and normalize_choices (choices.py) are consumed heavily by the ORM for making field definitions, constraints, and query lookups hashable and canonical. `[llm-analyzed]`
- **Middleware**: CSP utilities (csp.py) are consumed by SecurityMiddleware. LazyNonce is attached to requests; build_policy() is called per-response to render the final CSP header, replacing NONCE sentinels. `[llm-analyzed]`
- **Core / exceptions**: SynchronousOnlyOperation is raised by async_unsafe. SuspiciousOperation is raised by archive path traversal detection. These exceptions propagate to Django's core exception handling. `[llm-analyzed]`
- **Templates / templatetags**: Translation lazy strings (Promise subclasses) and format utilities (date_format, number_format) are used extensively in template rendering. The distinction between smart_str and force_str matters here — templates resolve lazy strings at render time. `[llm-analyzed]`

## Gotchas and edge cases
- Changing USE_I18N after any translation function has been called has no effect — the resolved backend is cached as an instance attribute on the singleton `_trans` object. Tests that toggle USE_I18N must manually clear `_trans.__dict__`. `[llm-analyzed]`
- CSP.NONCE in a policy config silently disappears if no nonce is passed to build_policy(). There is no warning or error. `[llm-analyzed]`
- get_format() silently returns the format_type string itself for unknown format types not in FORMAT_SETTINGS — passing a typo'd format name will not raise an error. `[llm-analyzed]`
- lazy_annotations() is not reentrant — concurrent use from multiple threads while holding the lock will deadlock. It patches a private CPython internal. `[llm-analyzed]`
- Archive extraction strips leading directory components if all archive members share a common prefix directory — this is intentional normalization but may surprise callers expecting paths to match the archive structure exactly. `[llm-analyzed]`
